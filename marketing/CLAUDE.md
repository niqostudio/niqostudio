# CLAUDE.md — marketing

NIQO STUDIO のマーケティング資産と投稿生成の正典。声・世界観・戦略・投稿済み履歴を1か所に集約し、投稿の下書き生成に使う。

## 役割と境界

- このモジュールが持つのは **NIQO STUDIO 水準の再利用資産**：[brand/](brand/)（声・世界観・ペルソナ）、[strategy/](strategy/)（投稿戦略）、[history/](history/)（投稿済み記録）。
- **生成の実体（コマンド）と下書きはここに置かない。** 各プロダクトのリポジトリ（非公開）側にコマンドを置き、ここの資産を読み込んで下書きをそのリポ側に出力する。
- 理由：このリポジトリは public。未公開プロダクトの素材・下書きが混ざらない構造で分離する。

## public リポ衛生（厳守）

- root の [conventions](../.claude/rules/conventions.md) に従い、**未公開プロダクトの名前・コードネーム・固有機構名を書かない**（brand・strategy・history・例示すべて）。
- 表記は **「NIQO STUDIO」で統一**（略称の単独使用はしない）。
- [history/](history/) に記録するのは **投稿済み（＝公開済み）の本文のみ**。下書きは非公開側に置く。

## 生成のワークフロー（各プロダクトリポのコマンドが従う）

1. 公開資産を読む：`brand/**`・`strategy/posting-strategy.md`・`history/**`。
2. 非公開のプロダクト素材を読む（各リポの `marketing/source.md`）。
3. 投稿戦略の「層1（問題・思想／製品を主張しない）」で下書きを生成。履歴と重複しない柱を選ぶ。
4. 下書きは非公開側（各リポの `marketing/drafts/`）に出す。**生成出力に仮名・固有機構名を含めない。**
5. 投稿後、その本文を [history/](history/) に記録する。
