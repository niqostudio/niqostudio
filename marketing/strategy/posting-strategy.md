# posting-strategy — X・note 投稿戦略

NIQO STUDIO の X・note の投稿方針。未公開プロダクトには触れず、問題・思想の発信（層1）を主に扱う。

## 発信の二層化

- **層1：問題・思想の発信（製品を主張しない）→ 先行して実施。** 目的は観客形成と「どの切り口が刺さるか」の検証。vaporware の予告はしない。
- **層2：製品の発信 → 製品が動いてから。** 詳細は各プロダクト側（非公開）で扱う。本書が直接対象とするのは層1。

## チャネルの役割分担

| | X | note |
| --- | --- | --- |
| 形式 | 短文・高頻度・日々の存在感 | 長文・低頻度・正典エッセイ |
| 役割 | 流通・観客形成・安い検証・議論への参加 | 世界観の正典化・信用・evergreen・被リンク先 |
| 関係 | スレッドから note エッセイへ誘導 | X の各主張の根拠になる |

## コンテンツの柱（層1）

- **柱1：AI 時代の保守・理解負債** — 注目が集まっている論点への観察とテイク。注目を借りる。
- **柱2：受託／業務委託のリアル** — 引き継ぎ・終了報告・スコープ説明・ドキュメントのドリフト。実案件由来の一次体験で語る。最も真似されにくい本丸。
- **柱3：思想** — [worldview](../brand/worldview.md) のテーゼ（what の液体化／why の揮発と資産化／build→operate）。
- **柱4：build-in-public** — 層2でのみ解禁。今は出さない。

## 立ち上げシーケンス（投稿ゼロからの最初の数手）

1. プロフィール（bio）整備（X・note）。製品名でなく **世界観**を掲げる。
2. note 第1エッセイ＝旗のテーゼ。固定（ピン）する。
3. X 固定スレッド＝上記の要約（5〜8ポスト）。最後に note へ誘導。固定する。
4. 以降、柱1〜3を巡回。
5. 放送より先に会話：既存の議論にリプライ・引用で参加し、発見されにいく。

## カデンス（受託業務と両立できる現実的な頻度）

- X：週3〜5ポスト、1〜2週に1スレッド。
- note：月1〜2本。1本を深く。
- 再利用：note エッセイ1本 → X スレッド＋複数の単発に分解。

## 観測するシグナル（検証として）

- **誰が反応するか**（[audience](../brand/audience.md) の層か）。バニティな数でなくターゲット層の反応を見る。
- **どの柱・切り口が最も刺さるか**。次の発信と重心の判断に反映する。
- **「これ欲しい」「うちも同じ痛み」** の声。個別の対話候補として記録する。

## アンチパターン

- 未公開プロダクトの名前・固有機構名を出す。
- 中身のない予告・誇張。
- 定型的・量産的な発信。
- 頻度を盛りすぎて途切れる。
